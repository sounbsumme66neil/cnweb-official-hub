# cnweb-official-hub

本仓库用于归档和发布多个独立 HTML 页面。这些页面不隶属于任何特定网站或域名，仅作为静态 HTML 资源的集中存放与版本管理空间。

## 项目简介

cnweb-official-hub 是一个通用型 HTML 页面仓库。所有页面均为独立文件，可直接在浏览器中打开或部署至任意静态托管服务。仓库本身不提供页面索引、导航或内容聚合功能，仅负责文件的存储与历史版本保留。

## 目录说明

```
/
├── pages/           # 存放所有 HTML 页面文件
├── assets/          # 页面引用的静态资源（如图片、样式、脚本）
├── archive/         # 历史版本或不再维护的页面
└── README.md        # 本文件
```

- `pages/`：当前活跃的 HTML 页面，按功能或主题命名。
- `assets/`：页面所需的公共或私有资源文件。资源文件建议与页面文件名对应，以便管理。
- `archive/`：已停止更新或不再使用的页面，移至此处以保持主目录整洁。

## 页面归档说明

1. **文件命名**：页面文件名应简洁且具备自解释性，避免使用特殊字符或空格。推荐格式：`项目名-功能.html`。
2. **元数据**：每个 HTML 文件应在 `<head>` 内包含 `<meta charset="UTF-8">` 和 `<title>` 标签，以便正确渲染。
3. **独立性**：每个页面应尽量自包含，避免依赖外部不可控资源。若需引用本地资源，请统一放置于 `assets/` 目录。
4. **版本管理**：对页面进行重大更新时，可将旧版本移入 `archive/` 目录，并保留原始文件名或添加时间戳后缀。

## 维护说明

- 本仓库由维护者根据需求不定期更新。
- 欢迎通过 Issue 提交页面新增或修改建议，但请注意：本仓库不接受与具体商业网站、推广内容或违规信息相关的页面。
- Pull Request 请确保页面内容完整、无外部恶意依赖，并遵循上述命名与目录规范。
- 仓库不保证页面长期可用性，也不承担任何因页面内容或使用方式引发的责任。
